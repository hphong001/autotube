폴더 구조 설정
AutoTube Pro의 모든 자동화 기능은 정해진 폴더 구조를 기반으로 동작합니다.
작업을 시작하기 전에 아래 순서대로 폴더를 설정하세요.
설치 순서
1
제공된 패키지 파일 다운로드
  • AutoTube Pro 구매 후 제공되는 패키지 파일(.zip)을 다운로드합니다.
2
C: 드라이브에 압축 해제 ★ 위치 필수
  • 다운로드한 파일을 반드시 C: 드라이브 최상단에 압축 해제합니다.
  • 압축 해제 후 C:\유튜브_영상제작\ 폴더가 생성됩니다.
3
새 영상 작업 시 프로젝트 폴더 추가
  • 영상_작업파일\ 안에 새 프로젝트명으로 폴더를 만들고 하위 3개 폴더를 그대로 복사합니다.
  • 예: 어닝시즌에살아남기\ → 금리인하의비밀\
폴더 구조
💻 C:\
📁 유튜브_영상제작\ ← 반드시 C: 드라이브에 위치
📁 영상_작업파일\
📁 어닝시즌에살아남기\ ← 프로젝트명 (자유롭게 설정)
📂 1_대본\ STEP 1 · 2 엑셀 대본 파일 보관
📂 2_이미지\ STEP 3 자동 생성 이미지 저장
📂 3_오디오_자막\ STEP 4 · 5 음성·자막 파일 저장
폴더관련 STEP자동 저장되는 파일
📂 1_대본\STEP 1STEP 2엑셀 대본 파일 (.xlsx)
📂 2_이미지\STEP 301.png, 02.png, 03.png …
📂 3_오디오_자막\STEP 4STEP 5001.wav … 합본.wav, 자막.srt
핵심설명
C: 드라이브가 아닌 다른 위치에 설치하면 자동화가 동작하지 않습니다.
새 영상 작업 시 프로젝트 폴더(예: 어닝시즌에살아남기)만 새로 만들면 됩니다. 하위 3개 폴더 구조는 동일하게 유지합니다.
각 STEP에서 경로를 지정할 때 해당 폴더를 직접 선택하면 됩니다.

각 번호별 기능 설명
1
대본 시트 — 메인 대본 내용 구성
  • 영상의 메인 대본 내용이 담긴 기본 시트입니다.
  • 사용자가 관리하는 내용으로 자유롭게 구성하시면 됩니다.
  • 단, 자동 이미지 생성을 위해 "이미지 생성 프롬프트" 열은 반드시 포함되어야 합니다.
2
"이미지 생성 프롬프트" 열 제목 ★ 필수
  • 이미지 자동 생성 시 프로그램이 이 열에서 프롬프트를 읽습니다.
  • 열의 위치는 어디든 상관없지만, 열 제목은 정확히 "이미지 생성 프롬프트"여야 합니다.
3
성우녹음 시트 ★ 필수
  • TTS 오디오 생성 시 이 시트의 텍스트를 읽어 음성을 생성합니다.
  • 시트 이름은 반드시 "성우녹음" 4글자여야 합니다.
  • TTS 오디오를 생성할 열 이름은 반드시 "성우녹음대본"으로 설정하세요.
  • 제공되는 AI성우_녹음대본_생성프롬프트.txt 파일을 참고하세요.
핵심설명
시트 이름은 반드시 "성우녹음" — 글자 수·띄어쓰기 모두 정확해야 합니다.
열 제목은 반드시 "이미지 생성 프롬프트" — 유사 표현은 인식 불가입니다.
위 조건 미충족 시 이미지·오디오 생성이 전혀 동작하지 않습니다.

  • 전체 / 현재 진행 / 남은 작업 / 진행률(%)을 실시간 표시합니다.
5
실시간 이미지 미리보기
  • 생성 완료된 이미지가 하단에 즉시 표시됩니다. 품질을 실시간으로 확인하세요.

생성된 이미지는 2_이미지\ 폴더에 자동 저장됩니다. 별도 파일명 변경이 필요 없습니다.

STEP 4 시작 전 — 기본 개념 이해
🔊
TTS란 무엇인가요?
TTS(Text-to-Speech)는 텍스트를 음성으로 변환하는 기술입니다. AutoTube Pro는 Qwen3-TTS 모델을 사용하여 제공하신 목소리 샘플과 거의 동일한 억양·톤으로 대본을 읽어줍니다.
☁️
왜 Google Colab을 사용하나요?
Qwen3-TTS 모델은 고품질 음성을 만들기 위해 강력한 컴퓨팅 자원이 필요합니다. Google Colab은 구글이 무료로 제공하는 클라우드 서버입니다. 내 PC 성능에 부담을 주지 않으면서 고품질 TTS를 무료로 사용할 수 있습니다.
🔗
전체 동작 방식
Colab에서 TTS 서버를 실행하면 gradio.live 웹 주소가 생성됩니다. AutoTube Pro가 이 주소로 대본 텍스트를 전송하면 Colab 서버가 음성을 생성해서 돌려보냅니다. 생성된 TTS 음성은 자동으로 내 PC에 저장 됩니다.
AutoTube Pro → 대본 텍스트 전송 → Colab 서버 (음성 생성) → AutoTube Pro 저장
1) AutoTube Pro — 오디오 준비 설정
오디오 생성 전 AutoTube Pro에서 먼저 필요한 정보를 설정하는 단계입니다.
시작전에 10초 정도의 깨끗하게 녹음된 자신의 목소리를 WAV 파일로 준비 합니다.
▲ [2] 오디오 자동 생성 패널 — 상단 설정 영역 (그림5)
각 번호별 기능 설명
1
TTS 생성에 필요한 학습용 목소리 샘플 파일을 선택 합니다.
2
샘플 파일 녹음 내용이 기입된 텍스트 파일을 선택 합니다.
3
현재는 한글만 지원 합니다.
4
TTS로 생성할 대본이 저장된 엑셀 파일을 선택 합니다.
5
TTS로 생성할 대본행의 번호를 지정 합니다.(자동 생성 범위)
6
개별 생성이 필요할때만 체크 표시를 ON하여 사용 합니다.
대본의 행 번호를 콤마(,)로 구분해서 입력 합니다.
7
위의 내용을 모두 설정하신 후 TTS 서버를 열어 녹음 준비를 합니다.
버튼을 누른 후 아래의 서버 설정 설명을 참고 하시기 바랍니다.
2) Google Colab — TTS 서버 가동
Google Colab은 구글이 제공하는 무료 클라우드 컴퓨팅 환경입니다.
내 PC 성능에 영향을 주지 않고 TTS(음성 합성) 서버를 무료로 가동할 수 있습니다.
▲ Google Colab — Qwen3-TTS 서버 가동 화면 (그림6)
각 번호별 기능 설명
1
"TTS 서버 열기 버튼"을 누르면 자동으로 이 화면이 열립니다.
2
TTS 서버를 가동할 시스템을 선택 해야 됩니다.
수정 -> 노트설정 -> T4 GPU -> 적용 순으로 선택 합니다.
3
화살표를 눌러 서버 기동 명령을 실행 합니다.
4
10초~수십초 후 설치 완료 메세지 출력이 나오는지 확인 합니다.
5
화살표 실행 버튼을 눌러 서버에 AI TTS를 실행 시킵니다.
6
서버가 준비될때 까지 기다립니다. (1~3분)
대기후 URL이 출력되면 서버 준비가 완료된 것입니다.
2번째 서버접속용 URL을 마우스를 이용해 정확하게 복사 합니다
3) AutoTube Pro — 오디오 녹음 실행
Colab 서버 가동 후 AutoTube Pro로 돌아와 오디오 녹음을 실행합니다.
▲ [2] 오디오 자동 생성 패널 — 녹음 실행 영역
각 번호별 기능 설명
1
위에서 복사한 TTS 서버 접속용 주소를 입력 합니다.
참고) 서버 생성시 마다 매번 변경 됩니다.
2
모든 설정이 완료된후 녹음을 시작 합니다.
(녹음 범위 설정후 자동생성 또는 개별생성 녹음)
3
녹음중 중지하는 기능이며, 중지후 이어서 녹음이 안되니 참고 하세요.
4
엑셀 대본의 모든 행은 개별 파일로 녹음됩니다.
자막 생성을 위해 1개의 파일로 합치는 기능 입니다.
핵심설명
목소리 원본과 복사할 내용을 정확히 입력해야 자연스러운 억양이 생성됩니다.
모든 컷 녹음 완료 후 반드시 "녹음 전체 합치기"를 실행합니다.
이 버튼을 누르지 않으면 STEP 5 자막 생성이 불가합니다.
Colab 세션 만료 시 오류 발생 → 셀2를 재실행하여 새 주소를 발급받아 붙여넣으세요.
🚨 [중요] TTS 서버 수동 종료 필수 안내
오토튜브는 사용자의 연속적인 작업을 고려하여 서버를 자동으로 종료하지 않습니다.
따라서 TTS 생성 완료 후 또는 더 이상 사용하지 않을 때는 반드시 직접 서버를 종료해 주시기 바랍니다.
(무료 사용자: 사용 제한 불이익 방지 / 유료 사용자: 불필요한 비용 차감 방지)

👉 종료 방법: 상단 메뉴 [런타임] ➔ [런타임 연결 해제 및 삭제]
STEP 5
자막 자동 생성
SRT 추출 · 타임코드 동기화 · 3_오디오_자막 폴더 저장
자막 자동 생성 & 타임코드 동기화
리소스 제작 탭 → [3] 자막 자동 생성 및 보정 섹션입니다.
합본.wav 오디오를 AI가 분석하여 각 컷의 정확한 시작·끝 시간을 추출하고,
SRT 자막 파일과 엑셀 타임스탬프를 자동으로 생성합니다.
▲ [3] 자막 자동 생성 및 보정 패널 — 33% 진행 중 (그림7)
각 번호별 기능 설명
1
원본 대본의 엑셀 파일을 선택 합니다.
2
자막을 생성할 최종 TTS 파일 *_합본.wav 파일을 선택 합니다.
3
만약 영상 검증 화면에서 자막 오류를 체크했다면 그 번호가 출력 됩니다.
4
자동으로 자막을 생성하고, 엑셀 파일에 각 행별 자막 출력 시간을 자동으로 입력 합니다.
STEP 6
검증 & 완성
통합 QC 검수 · 문제 컷 재작업 · CapCut 1초 조립
영상 리소스 검증 (QC)
영상 검증 탭입니다.
이미지·오디오·자막을 한 화면에서 동기화하여 개별적 또는 순차적으로 재생하며 품질을 검수합니다.
문제가 있는 컷만 체크박스에 마킹하고, 해당 STEP으로 돌아가 개별 재작업하면 됩니다.
▲ 영상 검증 탭 — 통합 QC & CapCut 조립 화면 (그림8)
검수 시 확인해야 할 항목
✓
이미지 확인
이미지가 해당 대본 내용과 맞는가 / 품질이 양호한가?
✓
오디오 확인
발음이 자연스러운가 / 잡음이나 끊김이 없는가?
✓
자막 확인
자막 타이밍이 오디오와 일치하는가 / 텍스트가 정확한가?
각 번호별 기능 설명
1
영상 검증 메인 탭
  • 상단 탭 바에서 영상 검증 탭을 선택합니다.
2
대본 불러오기
  • 엑셀 파일 경로 입력 후 대본 불러오기를 클릭합니다.
  • 각 컷의 대본·자막·썸네일·오디오 재생 버튼이 그리드에 표시됩니다.
  • 이미지 누락 컷 → 빨간 "없음" 뱃지 / 자막 누락 컷 → "자막없음" 뱃지로 즉시 식별 가능합니다.
3
개별 행별로 이미지 더블클릭, 오디오 듣기를 통해 검수 할 수 있으며,
재작없이 필요한 부분은 체크박스 체크하여 이후 개별 작업 할 수 있습니다.
4
5번의 재생버튼으로 재생할 범위를 지정 합니다.
5
순차적으로 자동 재생을 시작하는 합니다.
6
재생을 멈춘후 처음부터 다시 재생을 원할때 초기화 버튼을 누릅니다.
7
해당 체크박스를 선택한후 버튼을 누르면 체크된 내용을 초기화 합니다.
핵심설명
문제 발견 시 해당 컷의 오른쪽 체크박스 클릭 → 분홍색으로 마킹 → 해당 STEP으로 돌아가 개별생성으로 재작업합니다.
전체 공정을 처음부터 재실행할 필요가 없습니다. 문제 컷만 선택적으로 재작업하면 됩니다.
CapCut 완성 타임라인
캡컷 프로젝트 자동 생성이 완료되면 CapCut을 실행합니다.
수십 개의 이미지 클립·오디오·SRT 자막이 정확한 타이밍에
배치된 완성형 타임라인을 바로 확인할 수 있습니다.
▲ 캡컷 프로젝트 1초 생성
각 번호별 기능 설명
1
대본 엑셀 파일을 선택 합니다.
2
갭컷 프로젝트가 생성되는 경로 입닌다.
참고만 하시고 따로 설정하실 필요 없습니다.
3
16:9 롱폼영상 또는 9:16 쇼츠 영상등, 제작 하실 영상의 종류를 선택 합니다.
4
버튼을 누르는 순간 1초만에 캡컷 프로젝트가 생성 됩니다.
생성된 이름을 확인하여 캡컷에서 마무리 작업을 하시면 됩니다.
▲ CapCut — 자동 조립 완성 타임라인 화면 (그림9)
핵심설명
이미지 클립이 오디오 싱크에 맞춰 자동 배치되고, SRT 자막도 정확한 타임코드에 삽입된 상태입니다.
이후 작업은 트랜지션 효과, 배경음악 레벨, 브랜딩 요소 등 연출 디테일만 마무리하면 됩니다.
